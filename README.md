# C++ 编程项目集合

---

## 项目概述（Project Description）

### 项目一：TFTP 文件传输系统

**目录**: `TFTP/`

#### 项目描述
基于 UDP 协议实现的 TFTP (Trivial File Transfer Protocol) 文件传输系统，包含客户端和服务器两部分。

#### 主要特性
- ✅ 支持文件上传（WRQ - Write Request）
- ✅ 支持文件下载（RRQ - Read Request）
- ✅ 基于 UDP 的可靠传输机制
- ✅ 数据包分块传输（516 字节缓冲区）
- ✅ 错误处理和异常管理

#### 技术栈
- **协议**: UDP
- **标准**: C++11
- **构建工具**: Makefile

#### 应用场景
适用于简单的文件传输需求，如网络设备配置文件的上传下载、固件更新等。

---

### 项目二：线程池（Thread Pool）

**目录**: `threadPool/`

#### 项目描述
实现了一个通用的线程池框架，用于高效管理和调度多线程任务。

#### 主要特性
- ✅ 动态任务队列管理
- ✅ 基于条件变量的线程同步
- ✅ 支持任意可调用对象（函数、Lambda 表达式等）
- ✅ 自动线程生命周期管理
- ✅ 线程安全的任务添加和执行

#### 技术栈
- **标准**: C++11
- **核心技术**: `std::thread`, `std::mutex`, `std::condition_variable`, `std::queue`
- **构建工具**: Makefile

#### 架构设计
- 使用生产者-消费者模式
- 工作线程池预先创建，等待任务分配
- 任务队列采用 FIFO 策略

#### 应用场景
适用于需要并发处理大量独立任务的场景，如 Web 服务器请求处理、批量数据处理等。

### 项目三：网络聊天室（Chat Room）

**目录**: `chat/`

#### 项目描述
基于 TCP 协议的多客户端网络聊天室系统，支持实时消息广播和用户管理。

#### 主要特性
- ✅ 基于 TCP 的可靠连接
- ✅ 多客户端并发连接支持
- ✅ 消息广播功能
- ✅ 用户登录/退出管理
- ✅ 内置线程池处理客户端请求
- ✅ 消息序列化/反序列化

#### 技术栈
- **协议**: TCP
- **标准**: C++11
- **核心技术**: Socket 编程、多线程、消息序列化
- **构建工具**: Makefile

#### 消息类型
- `LOGIN (1)`: 用户登录
- `CHAT (2)`: 聊天消息
- `QUIT (3)`: 用户退出

#### 应用场景
适用于即时通讯、在线协作、实时消息推送等场景。

### 项目四：学生成绩管理系统（Student Manager）

**目录**: `studentManager/`

#### 项目描述
基于 SQLite 数据库的学生信息管理系统，提供完整的 CRUD 操作。

#### 主要特性

- ✅ 删除学生信息
- ✅ 修改学生信息
- ✅ 查询所有学生信息
- ✅ 查询特定学生信息
- ✅ 数据持久化存储（SQLite）

#### 技术栈
- **语言**: C (C99)
- **数据库**: SQLite3
- **构建工具**: Makefile

- 数据库文件: `student.db`

### 项目五：电子词典（Dictionary）

**目录**: `dictionary/`

#### 项目描述
基于客户端-服务器架构的网络电子词典系统，结合 SQLite 数据库和 TCP 网络编程。

#### 主要特性
- ✅ 用户注册与登录
- ✅ 单词查询功能
- ✅ 查询历史记录
- ✅ 用户在线状态管理
- ✅ 多客户端并发支持
- ✅ 数据库持久化存储

#### 技术栈
- **协议**: TCP
- **标准**: C++17
- **数据库**: SQLite3
- **核心技术**: 多线程、网络编程、数据库操作
- **构建工具**: CMake (2.8+/3.12+)

#### 架构设计
- **服务器端**:
  - 数据库管理器（`DatabaseManager`）：管理用户数据库和词典数据库
  - 服务器（`Server`）：处理客户端连接和请求
  - 多线程并发处理客户端请求
- **客户端**:
  - 网络通信模块
  - 用户交互界面

#### 消息类型
- `R (1)`: 注册
- `L (2)`: 登录
- `Q (3)`: 退出
- `S (4)`: 查询单词
- `H (5)`: 查看历史记录

#### 数据库文件
- `usr.db`: 用户信息数据库
- `dict.db`: 词典数据库

#### 应用场景
适用于在线词典服务、语言学习平台等场景。

### 项目六：数据库连接池（Connection Pool）

**目录**: `coonnectionPool/`

#### 项目描述
实现了一个高性能的 MySQL 数据库连接池，用于优化数据库连接的创建和管理。

#### 主要特性
- ✅ 连接池初始化和动态扩展
- ✅ 连接复用机制
- ✅ 空闲连接自动回收
- ✅ 连接超时管理
- ✅ 线程安全的连接获取和释放
- ✅ 基于配置文件的参数管理

#### 技术栈
- **标准**: C++11
- **数据库**: MySQL
- **核心技术**: `std::shared_ptr`, `std::mutex`, `std::condition_variable`, `std::atomic`
- **构建工具**: Makefile

#### 架构设计
- **连接管理**: 使用队列管理空闲连接
- **生产者线程**: 动态创建新连接
- **扫描线程**: 定期清理超时的空闲连接
- **智能指针**: 使用 `shared_ptr` 自动管理连接生命周期

#### 配置参数
- `ip`: 数据库服务器地址
- `port`: 数据库端口
- `username`: 数据库用户名
- `password`: 数据库密码
- `dbname`: 数据库名称
- `initSize`: 初始连接数
- `maxSize`: 最大连接数
- `maxIdleTime`: 最大空闲时间
- `connectionTimeOut`: 连接超时时间

#### 应用场景
适用于高并发 Web 应用、数据密集型应用等需要频繁数据库操作的场景，可显著提升数据库访问性能。
